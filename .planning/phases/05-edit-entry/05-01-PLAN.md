# Phase 5 — Plan 01: 追番编辑页 + 工具编辑页

## Goal

创建两个独立编辑页，维护者在 `/admin` 工作台点击条目卡片后跳转至此，表单回填当前全部可编辑字段，修改后保存至 localStorage 并返回工作台。

## Verification

1. `npm run build` exit 0（目录式动态段 `[slug]` 生成正确、define:vars 序列化正常）
2. `npx astro check` exit 0
3. 浏览器走查：
   - `/admin/edit/anime/葬送的芙莉莲` — SSR 回填所有字段，status select 正确选中 "done"
   - 修改 status 为 "watching"，progress 改为 20，保存后 localStorage 有对应 key
   - myRating 输入 15 → 红色高亮 + 错误提示，保存按钮禁用
   - cover 为空时显示 "当前封面为本地图片，编辑时自动保留原值"
   - `/admin/edit/tools/Raycast` — SSR 回填全部字段，tags chip 显示可删，category select 正确选中
   - body textarea 输入内容 → 下方预览区实时渲染 Markdown
   - 保存后跳转 `/admin`，取消按钮也跳转 `/admin`（不保存）

## Context / Patterns

C-01 目录式动态段 `[slug]/index.astro` 模式适配 Windows（`src/pages/admin/edit/anime/[slug]/index.astro`）。参考 `src/pages/anime/[slug]/index.astro` 的目录结构。

C-02 SSR 序列化到 `define:vars` 的数据不得包含不可序列化对象（`image()`、`render`、`Date`）。字段映射见 CONTEXT.md「Canonical References」。

C-03 两个编辑页共享的 localStorage key：
```typescript
const EDIT_KEY = 'chuxi_admin_edit_v1';
// 存：
// { 'anime:葬送的芙莉莲': { fields... }, 'tools:Raycast': { fields... } }
```

C-04 编辑页加载时先检查 localStorage 是否有该条目的草稿，如有则用草稿预填（而非 SSR 原始值），支持刷新后恢复编辑状态。

C-05 marked CDN: `https://cdn.jsdelivr.net/npm/marked/lib/marked.umd.min.js` — 仅在 tools 编辑页加载。

C-06 两个页面不需要 Preact / 框架 — 延续 Phase 4 原生 `<script>` 岛屿模式。

**action:**

新建 `src/pages/admin/edit/anime/[slug]/index.astro`。

**Frontmatter：**
1. import BaseLayout / Header / Footer / Icon
2. `const { slug } = Astro.params;` 获取 slug
3. `const allAnime = await getAllAnime();` 遍历找 slug 匹配
4. 匹配不到时用 `Astro.redirect('/admin', 302)` 或显示 404
5. 构造序列化数据：
```typescript
const entry = allAnime.find((a) => slugOf(a.data.titleCn) === slug);
if (!entry) return Astro.redirect('/admin');

const editData = {
  id: entry.id,
  slug,
  type: 'anime',
  fields: {
    titleCn: entry.data.titleCn,
    titleJa: entry.data.titleJa || '',
    status: entry.data.status,
    progress: entry.data.progress,
    episodes: entry.data.episodes ?? '',
    myRating: entry.data.myRating ?? '',
    comment: entry.data.comment || '',
    cover: typeof entry.data.cover === 'string' ? entry.data.cover : '',
    airDate: entry.data.airDate ? String(entry.data.airDate).slice(0, 10) : '',
  },
  hasLocalCover: typeof entry.data.cover !== 'string' && !!entry.data.cover,
};
```

**Body：**
- `<BaseLayout title="编辑追番 - 管理后台" description="编辑追番条目">`
  - `slot="head"`: `<meta name="robots" content="noindex, nofollow" />`
  - `<Header />`
  - `<main>` 容器 `mx-auto max-w-2xl px-4 py-12`
    - 返回链接：`<a href="/admin" class="...">&larr; 返回工作台</a>`
    - 标题：`<h1>编辑追番：{entry.data.titleCn}</h1>`
    - `<form id="edit-form">`：
      - titleCn: 只读显示（不可改，slug 依赖它）
      - status: `<select>` options(watching/done/plan) + `aria-describedby` 错误容器
      - progress: `<input type="number" min="0">` + episodes 显示 "集/共 N 集"
      - myRating: `<input type="number" min="0" max="10" step="1">` + 提示 "0-10"
      - comment: `<textarea rows="3">`
      - cover: `<input type="text" placeholder="https://...">` — 若 `hasLocalCover` 则在输入框上方显示灰字提示
    - 保存/取消按钮组

**客户端 `<script define:vars={{ editData }}>`**：
- 草稿恢复：读取 `chuxi_admin_edit_v1`，如有该条目数据则用草稿值覆盖 SSR 值
- 校验逻辑（`submit` 事件 + `input` 事件实时校验）：
  - myRating: `val === '' || (val >= 0 && val <= 10 && Number.isInteger(Number(val)))`
  - progress: `val === '' || (val >= 0)`
  - 校验失败：输入框加 `class="error"` + 显示下方错误文字
  - 任一校验失败：禁用保存按钮
- 保存处理：组装 fields 对象 → `chuxi_admin_edit_v1` ← merge 写入 → `window.location.href = '/admin'`
- 取消按钮：`window.location.href = '/admin'`
- 初始化恢复草稿后触发一次校验

## Risks / Decisions

- **marked CDN 加载**：tools 编辑页动态加载 `marked`，需确保 CDN 可用。退路：预览区显示"预览加载中"然后转为纯文本显示。
- **编辑页刷新后草稿恢复**：脚本初始化时先读 `chuxi_admin_edit_v1`，如有该条目数据则覆盖 SSR 值。用户保存后该 key 更新。
- **Bracket 文件名**：Windows 上 `[slug]` 目录名需确认 Astro 正确处理。
- **cover 字段序列化**：Astro Content Layer 将 `image()` 解析为 `ImageMetadata` 对象（typeof 'object'），`z.string().url()` 保持为字符串。用 `typeof entry.data.cover === 'string'` 区分。
