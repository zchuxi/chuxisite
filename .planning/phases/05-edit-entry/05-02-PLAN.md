**Part B — 脚本内增加「已编辑」逻辑**

在 `<script define:vars={{ adminItems }}>` 中：

1. 在脚本顶部常量区新增：
```javascript
const EDIT_KEY = 'chuxi_admin_edit_v1';
```

2. 在 `renderView()` 之后、初始化调用之前新增函数：
```javascript
function applyEdits() {
  const edits = JSON.parse(localStorage.getItem(EDIT_KEY) || '{}');
  Object.entries(edits).forEach(([key, edit]) => {
    // key format: "anime:葬送的芙莉莲" or "tools:Raycast"
    const [type, title] = [key.slice(0, key.indexOf(':')), key.slice(key.indexOf(':') + 1)];
    // 查找对应卡片
    const cards = document.querySelectorAll(`[data-type="${type}"]`);
    cards.forEach((card) => {
      const parent = card.closest('.relative');
      if (!parent) return;
      // 添加已编辑角标（右上角小铅笔图标）
      let badge = parent.querySelector('.edit-badge');
      if (!badge) {
        badge = document.createElement('div');
        badge.className = 'edit-badge';
        badge.innerHTML = '✎';
        badge.setAttribute('aria-label', '已编辑');
        parent.appendChild(badge);
      }
      // 更新卡片视觉：卡片标题改为最新值
      // 简单方案：在卡片容器上方叠加半透明覆盖层 + "已编辑"文字
      // 或更醒目方式：卡片整体加半透明覆盖 + 右上角小蓝点
      // 覆盖层点击仍进入编辑页（href 不变）
    });
  });
}
```

实际实现应简单可靠。推荐方案：
- 在卡片 `.relative` 容器右上角插入一个 `.edit-badge` 元素（z-30，与删除按钮不重叠）
- 卡片本身的标题/状态文字保持 SSR 原值（改 DOM 文本复杂且容易出错；让用户点击进入编辑页查看最新值）

3. 在初始化代码中 `renderView()` 后调用 `applyEdits()`：
```javascript
renderView();
applyEdits();  // 新增
getDeleted().forEach(...  // 原有
```
