# Phase 1 — UI Design Contract

> Visual and interaction contract for「站点地基与二次元外壳」. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Style: 鲜艳可爱型二次元（cute, vibrant anime）· 青蓝清爽主色 · 很圆润软萌 · 多二次元点缀（克制、不伤性能与可读性）。

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — shadcn 不适用（Astro 静态站，非 React/Next/Vite 组件体系） |
| Preset | not applicable |
| Component library | none — 纯 `.astro` 组件；交互仅在真正需要处用原生 `<script>`（主题切换），Preact 岛屿本阶段不需要 |
| Icon library | `astro-icon` + Iconify Phosphor（`@iconify-json/ph`）— 构建期内联 SVG、零运行时 JS、圆润友好，契合软萌基调 |
| Font (headings) | `Fredoka`（Latin，可变，圆润可爱）+ `ZCOOL KuaiLe`（CJK 圆润快乐体）作为回退链；子集化 + `preload` + `font-display: swap`；**仅用于标题/logo/少量点缀**（PITFALLS P5） |
| Font (body) | 系统字体栈：`system-ui, -apple-system, "PingFang SC", "Microsoft YaHei", "Segoe UI", sans-serif`（零加载成本、CJK 可读） |

Font-family tokens（Tailwind v4 `@theme`）:
```
--font-display: "Fredoka", "ZCOOL KuaiLe", system-ui, sans-serif;
--font-body: system-ui, -apple-system, "PingFang SC", "Microsoft YaHei", "Segoe UI", sans-serif;
```
Font strategy: 标题字体只加载 heading 权重，`<link rel="preload" as="font" crossorigin>` 关键字重；CJK 快乐体按站点实际用字子集化（pyftsubset），避免单文件 >1MB（P5 验收：网络面板无 >1MB 字体请求）。

## Spacing Scale

Declared values (multiples of 4, 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 图标与文字间距、行内微调 |
| sm | 8px | 紧凑元素间距、徽章内边距 |
| md | 16px | 默认元素间距、卡片内边距基线 |
| lg | 24px | 卡片内边距、组件块间距 |
| xl | 32px | 区块间距、卡片外间距 |
| 2xl | 48px | 主要区块分隔（Hero 与下方入口区） |
| 3xl | 64px | 页面级留白（桌面 section 上下） |

Exceptions: 交互触控目标最小 **44px**（主题切换按钮、图标按钮、导航链接命中区）— 满足移动端可点性（PITFALLS「移动端卡片过密」）。

## Typography

4 roles, **2 weights only**（400 regular / 600 semibold）:

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px | 400 | 1.5 | body（系统栈） |
| Label | 14px | 600 | 1.4 | body（徽章/标签/导航/按钮文字） |
| Heading | 24px | 600 | 1.25 | display（区块标题、卡片标题） |
| Display | clamp(32px, 6vw, 52px) | 600 | 1.15 | display（Hero 主标题，响应式收缩） |

规则：正文永不用装饰字体；Display/Heading 用 `--font-display`。小字号（≤14px）严禁花体（P5 / 可读性）。

## Color

60/30/10 分配，亮/暗双主题。所有 token 定义为 Tailwind v4 `@theme` 变量，暗色经 `@custom-variant dark (&:where(.dark, .dark *))` 覆盖（PITFALLS P3）。

### Template 映射

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) | `#F3FBFE` | `#0C1B22` | 页面背景 |
| Secondary (30%) | `#FFFFFF` | `#12262F` | 卡片、导航栏、页脚、面板 |
| Accent (10%) | `#17B6D6` | `#3AD2EF` | 见「Accent reserved for」 |
| Destructive | `#F26D6D` | `#F58686` | 破坏性操作（本阶段无，预留 token） |

### 完整调色板（`@theme` tokens）

| Token | Light | Dark | 说明 |
|-------|-------|------|------|
| `--color-bg` | `#F3FBFE` | `#0C1B22` | 主背景（60%）青蓝清爽底 |
| `--color-surface` | `#FFFFFF` | `#12262F` | 卡片/导航/页脚（30%） |
| `--color-surface-muted` | `#E7F6FB` | `#17303A` | 悬浮态、次级面板、代码块底 |
| `--color-primary` | `#17B6D6` | `#3AD2EF` | 主色青蓝（accent 10%）填充/激活 |
| `--color-primary-hover` | `#0FA0BE` | `#5CDCF2` | 主色悬浮态 |
| `--color-primary-fg` | `#FFFFFF` | `#04222B` | 主色填充上的前景文字 |
| `--color-link` | `#0A7C93` | `#6FE0F5` | 正文链接文字（对底 AA 合格，勿用 `--color-primary` 做白底小字） |
| `--color-text` | `#0F2E3A` | `#E6F6FA` | 正文主文本 |
| `--color-text-muted` | `#5A7A85` | `#94B4BF` | 次要文本、描述、页脚 |
| `--color-border` | `#CDEBF3` | `#234450` | 卡片/输入边框、分隔线 |
| `--color-success` | `#34C77B` | `#4BD68C` | 状态成功/在线（预留，追番状态色日后扩展） |
| `--color-destructive` | `#F26D6D` | `#F58686` | 破坏性（本阶段不用） |
| `--color-deco-pink` | `#FF8FC7` | `#FF9ECF` | **仅装饰**（星星/爱心/blob/手绘分隔）—— 非交互，绝不作可点信号 |

Accent reserved for: 主 CTA 按钮填充、激活/悬浮的导航链接、焦点环（focus ring）、正文链接（用 `--color-link`）、入口卡片图标圆底、主题切换激活态。**不用于**大面积背景、正文文本、装饰点缀。

对比度：`--color-text` 对 `--color-bg` 两主题均 ≥ AA 4.5:1。主色 `#17B6D6` 对白**不达标做小字**——白底文字链接一律用 `--color-link`；主色只作填充块 + `--color-primary-fg` 白字，或大号/边框。暗色同校验（PITFALLS「亮/暗切换只调背景」「荧光色牺牲可读性」）。

## Radius（很圆润软萌 — 锁定决策，核心视觉）

| Token | Value | Usage |
|-------|-------|-------|
| `--radius-sm` | 10px | 徽章、输入框、小标签 |
| `--radius-md` | 16px | 按钮 |
| `--radius-lg` | 24px | 卡片 |
| `--radius-xl` | 32px | Hero 容器、大入口卡片、插画占位 |
| `--radius-full` | 9999px | 图标按钮、头像、pill、主题切换 |

阴影：柔和低对比，青蓝染色而非纯黑。`--shadow-soft: 0 6px 24px -8px rgba(23,182,214,0.25)`（暗色降透明度至 ~0.15，避免发灰刺眼）。

## Layout & Component Inventory（供 planner/executor）

页面容器：内容最大宽 `72rem`（1152px）居中，左右安全内距 `md`(16px)→`lg`(24px)。响应式断点用 Tailwind v4 默认（sm 640 / md 768 / lg 1024 / xl 1280）。

| Component | 契约要点 |
|-----------|---------|
| `BaseLayout.astro` | `<head>`：内联阻塞主题脚本（首）、SEO meta、字体 preload；`slot` 主体；页脚。props: `title`/`description`/`ogImage?` |
| `Header/Nav` | sticky top-0，`--color-surface` + `backdrop-blur` + 底部 `--color-border`；含 Logo、导航链接（工具库/追番）、主题切换。链接命中区 ≥44px，激活态用主色下划线/圆点 |
| `Logo/Wordmark` | `--font-display` 站名 + 一个星星 ✦ 点缀（`--color-deco-pink`，`aria-hidden`）|
| `ThemeToggle` | `--radius-full` 图标按钮 44×44，太阳/月亮切换；原生 `<script>`，`aria-label="切换深浅色主题"` |
| `Hero` | 见下「Hero 规格」|
| `IntroBlock` | Hero 下方站点简介：Heading + 1–2 句 Body 文案，居中或左对齐，`2xl` 上间距 |
| `EntryCard`（×2） | 大入口卡；1 列（<md）→ 2 列（≥md），`gap: xl`。整卡为 `<a>`（可点区=整卡）。结构：图标圆底（`--radius-full`，主色 10% 染底）+ 卡片标题（Heading）+ 描述（Body/muted）+ 箭头。`--radius-xl`、`--shadow-soft`、hover 上浮 `translateY(-2px)`（≤180ms，`prefers-reduced-motion` 下禁用）。工具库卡视觉略重（核心价值：主色描边/更强染底） |
| `Footer` | `--color-surface`，muted 文案、版权、一句软萌签名；分隔用手绘 SVG divider（`aria-hidden`）|

| Decoration SVGs | 星星/爱心/圆点/blob，纯 SVG/CSS，`aria-hidden="true"`，绝对定位不占文档流、不遮挡文本、不拦截点击（`pointer-events:none`）|

## Hero 规格（用户后补插画 — 需优雅占位）

- **布局**：桌面（≥lg）两栏——左：Display 主标题 + 一句副标题（Body/muted）+ 主 CTA `进入工具库` + 次 CTA `查看追番`；右：插画。移动端（<lg）单栏纵向堆叠，插画在文字**下方**，宽度铺满容器但保持比例。
- **背景**：Hero 区全宽柔和渐变（`--color-primary` → `--color-deco-pink` 低透明），叠加少量漂浮星点装饰（`aria-hidden`）。
- **插画尺寸**：容器 `max-width: 520px`，固定 `aspect-ratio: 4/3`（用 `--radius-xl` 裁圆角）。走 `astro:assets` `<Image>`，显式 `width`/`height`，作为 LCP：`loading="eager"` + `fetchpriority="high"`，输出 WebP/AVIF（PITFALLS P4，CLS≈0）。
- **占位回退（无图时）**：同尺寸同 `aspect-ratio` 的圆角渐变 blob（primary→pink 青粉渐变），中心放 1–2 个星星/爱心 SVG + 轻微浮动动效（`prefers-reduced-motion` 下静止）。保证与真图同占位，切换不产生 CLS。占位不含实义文本、`aria-hidden`。

## Theme（亮/暗 · 记忆 · 无 FOUC）

- `<head>` **首个**内联阻塞脚本（非 module、无 `client:`）：`localStorage.theme ?? matchMedia('(prefers-color-scheme: dark)')` → 在绘制前给 `<html>` 加/去 `.dark`。
- 切换按钮原生 `<script>` 写 `localStorage.theme` 并 toggle `.dark`；图标随态切换。
- `astro:after-swap` 重新应用主题（若启用 View Transitions），防跳转丢主题。
- 验收：刷新暗色页无闪白、控制台无 hydration 警告（PITFALLS P3 / 「Looks Done But Isn't」）。

---

## Motion & Accessibility

- 动效克制：仅 `transform`/`opacity`/`color`，时长 ≤200ms；hover 上浮、图标过渡等非必要动效全部包裹 `@media (prefers-reduced-motion: reduce)` 关闭。
- 焦点可见：所有交互元素 `:focus-visible` 显示 2px `--color-primary` 焦点环 + 2px offset，圆角随元素。
- 装饰全部 `aria-hidden` + `pointer-events:none`，不进 Tab 序、不干扰读屏。
- 触控目标 ≥44px；正文两主题 WCAG AA ≥4.5:1。

## Meta / SEO Contract（INFRA-06）

- `astro.config` 设 `site`；集成 `@astrojs/sitemap` 构建期出 `sitemap.xml`。
- `BaseLayout` 每页输出：`<title>`（模式 `{页面标题} · 二次元小站`）、`<meta name="description">`、Open Graph（`og:title`/`og:description`/`og:image`/`og:type`/`og:url`）、`canonical`（由 `site` 拼接）。
- 默认 OG 图：站点级兜底图（含 logo/主色），页面未显式提供 `ogImage` 时使用。
