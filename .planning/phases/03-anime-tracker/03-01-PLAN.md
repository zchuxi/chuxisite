## Task 1 — 扩展 `src/lib/anime.ts`（slugOf + getAnimeByStatus）

### read_first

- 被改文件：`src/lib/anime.ts`
- Source of truth：`src/lib/tools.ts:1-33` — `slugOf`、`getToolsByCategory`、`getRelated` 的 Phase 2 已验证模式，本 plan 镜像。
- Source of truth：`src/content.config.ts:24-43` — anime schema（`status: z.enum(['watching','done','plan'])`、`titleCn: z.string()`、`draft` 过滤依据）。**本 plan 不修改此文件。**
- Source of truth：`.planning/phases/03-anime-tracker/03-CONTEXT.md` — D-09、D-01、D-10。
- Source of truth：`.planning/phases/03-anime-tracker/03-UI-SPEC.md` — Layout/Grid、组件消费字段。
- Seed shape：`src/content/anime/frieren.mdx` — `status/progress/myRating/episodes/comment/titleJa/titleCn` 字段形状。

### action

1. 在 `src/lib/anime.ts` 顶部新增导出：
   ```ts
   export function slugOf(titleCn: string): string {
     return encodeURIComponent(titleCn);
   }
   ```
   与 `lib/tools.ts:11-13` 完全一致。
2. 新增导出函数：
   ```ts
   export async function getAnimeByStatus(
     status: 'watching' | 'done' | 'plan',
     opts: { excludeSlug?: string } = {},
   ) {
     const all = await getAllAnime();
     return all.filter(
       (a) => a.data.status === status && slugOf(a.data.titleCn) !== opts.excludeSlug,
     );
   }
   ```
   要点：复用 `getAllAnime()` 的 draft 过滤 + watching→plan→done 排序，**不重复** `getCollection` 调用（收口点单一，日后换 Bangumi 仅改 `getAllAnime` 内部，ANIME-05）。
3. `getAllAnime()` 主体不动；`STATUS_ORDER` key 类型保持字面联合（已是）。
4. 不新增 `getRelatedAnime` — UI-SPEC 未要求「相关番剧」；YAGNI。

**ponytail**: 不复用 getCollection 直读；不预置 lru_cache；不新增 AnimeStatus 等冗余类型。`opts.excludeSlug` 单一可选参数，足矣。

### acceptance_criteria

- [ ] `src/lib/anime.ts` 导出 `getAllAnime`、`slugOf`、`getAnimeByStatus`，`npx tsc --noEmit astro` 0 错。
- [ ] `slugOf('葬送的芙莉莲')` 返回编码字符串（与 `encodeURIComponent('葬送的芙莉莲')` 相等）；浏览器态/CLI 快速验证即可，无单测框架依赖。
