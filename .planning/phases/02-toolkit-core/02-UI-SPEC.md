# Phase 2 — UI Design Contract（工具库核心）

> Visual and interaction contract for「工具库（核心）」. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> 范围：TOOL-01..05。在 Phase 1 视觉基线上扩展工具库主页、详情页、分类/标签档案页的全套视觉与交互契约。
> 本阶段**零 Preact 岛屿**（分类/标签走构建期静态页，主题沿用 Phase 1 原生 `<script>`）。

---

## Design System（继承 Phase 01，这里仅列出差异/新增）

Phase 01 的 Design System（配色、radius、shadow、字体、动效、a11y 基线）**全部直接复用、不再重定义**。详见 `.planning/phases/01-foundation-shell/01-UI-SPEC.md` § Design System / Spacing Scale / Typography / Color / Radius。下表仅做增量声明。

| Property | Value |
|----------|-------|
| Tool | none（同 Phase 01；纯 `.astro`，无 shadcn） |
| Component library | none — 复用既有 `EntryCard` / `Header` / `Footer` / `BaseLayout` / `deco/*` |
| Icon library | `astro-icon` + Iconify Phosphor（`ph:`）；本阶段新增 `ph:arrow-square-out`（外链按钮）、`ph:ph:tag`（标签）、`ph:folder-simple`（分类） |
| Font | `--font-display`（Fredoka + ZCOOL 回退）/ `--font-body`（系统栈）— 工具标题用 display，描述/正文用 body |
| Island framework | **无**。主题沿用 Phase 1 原生 `<script>`；工具库无客户端状态，不引 Preact / nanostores |

## Spacing Scale（完全复用 Phase 01）

| Token | Value | Phase 2 中的新用法 |
|-------|-------|--------------------|
| xs (4px) | 图标与文字间距、外链按钮内边距 | 外链按钮 `px-2.5 py-2.5` |
| sm (8px) | 紧凑元素间距 | 分类 pill / 标签 chip 之间的 gap |
| md (16px) | 默认元素间距、安全内距 | 页面左右 `px-4`、卡片内元素 |
| lg (24px) | 组件块间距 | 卡片内边距 `p-6`、分类 pill 行与网格间距 |
| xl (32px) | 卡片外间距、区块入口分隔 | 网格 `gap-6` → `gap-8(lg)`、主页各区块竖直间距 `mt-8/10/12` |
| 2xl (48px) | 主要区块分隔 | 工具条（summary line + 分类 pills）与标签云之间、标签云与网格之间 |
| 3xl (64px) | 页面级留白 | 同 Phase 1 Footer 上间距 `mt-24` 等价 |

Exceptions: 交互触控目标最小 **44px** (`min-h-11 min-w-11`)— 外链按钮、分类 pill、标签 chip、面包屑链接命中区。

## Layout & Grid（D-03 / D-13）

### 全局容器

与 Phase 01 一致：`max-w-[72rem]` 居中，左右安全内距 `px-4` → `md:px-6`。

### 卡片网格

| 断点 | 列数 | gap |
|------|------|-----|
| < md (默认) | 1 列 | `gap-6`（24px） |

实现：`grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6 lg:gap-8`。

### 主页垂直骨架（D-13，自上而下）

```
<BaseLayout>
  <Header />
  <main>
    [summary line]   "共 N 件工具 · 慢慢整理中"   ← 站点说明 + 计数，y-8
    [category pills]  分类入口 pill 行，可横滚（overflow-x-auto），y-8
    [tag cloud]      标签云 chip，松散 wrap，y-12
    [tool grid]      1→2→3 列卡片网格，y-12

  <Footer />
</BaseLayout>
```

- 区块间竖直 rhythm：summary→pills `mt-6`，pills→tag cloud `mt-8`，tag cloud→grid `mt-8`。
- 页面底部 Footer 前 `mt-24`（沿用 Phase 1）。

### 详情页垂直骨架

```
<BaseLayout>
  <Header />
  <main>
    [breadcrumb]      "工具库 / 工具名"   ← 顶部 y-4

```

## Component Inventory（每组件写清 states / 间距 / 响应式）

### `EntryCard.astro`（复用 + 改造，D-01 / D-04 / D-10）

Phase 2 把整卡从「入口大卡」泛化为「工具卡片」。改造点：

1. 新增 prop：`url?: string`（外链地址）、`external?: boolean`（默认 true，整卡 href 走详情；外链按钮走 url）。
2. 卡片内部新增**独立外链图标按钮**：
   - 位置：标题行 right 侧（覆盖原 `ph:arrow-right`），或卡片右下角。
   - 形态：Phosphor `ph:arrow-square-out`，`size-5`，色 `--color-text-muted` → hover `--color-primary`。
   - 可点区：`<a href={url} target="_blank" rel="noopener noreferrer" aria-label="{title} 外链" class="inline-flex min-h-11 min-w-11 …">`。
   - 事件：click 时 `e.stopPropagation()` + `e.nativeEvent.stopPropagation()`（Astro 输出原生 `<a>`，实际只需 stopPropagation，因为不是 JS 跳转；但 Astro 静态锚点无 JS，故**只在需要 prevent 整卡冒泡时添加**。更合适的做法是外链按钮设为独立 `<a>`，布局上用 `position` 覆盖在卡片右上角，此时整卡 `<a>` 不包住按钮即可避免冒泡；**推荐此方案**）。
   - TOOL-05 强制：`target="_blank" rel="noopener noreferrer"`。
3. 整卡 `<a>` href 指向详情页 `/tools/[slug]`（`slug` 派生见 D-09）。
4. 状态继承 Phase 1：`border` + `--color-border`，hover `--color-primary-hover` 描边 + `motion-safe:-translate-y-0.5` `duration-150`；`radius-xl` + `shadow-soft`。

States: default / hover（上浮 + 描边主色）/ focus-visible（2px primary ring + 2px offset）/ active（回位）。

响应式：1 / 2 / 3 列由父 grid 决定，卡片本身无宽度断点。

### Category pills（分类入口行，D-13）

- 横向排列，间距 `gap-2`；移动端 `overflow-x-auto` + `flex-nowrap` + `pb-1` 滚动保护，桌面端 `flex-wrap`。
- 每个 pill：`<a>` 类 `--radius-full`、`px-3.5 py-1.5`、`text-sm font-semibold`、`border` 用 `--color-border`、文字 `--color-text-muted`；hover 边框变 primary + 文字变 primary；激活态（当前分类页）填充 `--color-primary` + `--color-primary-fg`。
- 命中区 ≥44px (`min-h-11`)。
- 首项固定「全部工具」回到 `/tools`。
- 数据驱动：经 `getAllTools()` 去重 category 枚举。

States: default / hover / active(aria-current=page) / focus-visible。

### Tag cloud（标签云，D-13）

- 松散 wrap 布局，`flex flex-wrap gap-2`。
- chip 形态：`<a>` `--radius-full`、`px-2.5 py-1`、`text-xs font-semibold`、染底 `color-mix(in oklab, var(--color-primary) 8%, transparent)`、文字 `--color-text-muted`、`border-transparent`。hover 文字变 `--color-primary`。
- chip 前加 `#` 文本（`aria-hidden`），仅视觉。命中区 `<44px` 允许（云内 chip 小些可读，但需保证高度 ≥ 32px + 文字行高；若实测偏小可在 md+ 升到 36px）。
- 数据驱动：经 `getAllTools()` 展开 tags 去重（transform 归一化后）。
- 排序：按出现次数降序或按字母序（planner 自定；标注在 PLAN）。

States: default / hover / focus-visible。

### Breadcrumb（面包屑）

- `<nav aria-label="路径">` > `<ol>`，首页用 `/` 分隔（`aria-hidden`）。
- 结构：`工具库 › {分类?} › {当前}`。分类段可选。
- 链接色 `--color-link`，当前页（最后一项）`--color-text` 无链接。
- Meta 13px。
- 复用 `<Icon name="ph:caret-right" aria-hidden>` 作为分隔（比 `/` 更柔软）。

States: link 默认/hover/focus-visible；末项静止。

### 详情页 MDX 正文区

- 用 Astro `<Content />` 渲染 MDX。
- 容器 `max-w-[48rem]`（更窄阅读栏，保持 ~70 CPL）居中，`prose` 风格排版。
- 正文样式继承 body token；`h2/h3` 用 display；`code` 块 `--color-surface-muted` 底 + `--radius-sm`。
- 嵌入 `<Image>`：走 `astro:assets`，`--radius-lg` 圆角，`loading="lazy"` 非首图。
- 暗色主题：链接色用 `--color-link`，代码块底同 token。

### 详情页底部 nav（D-11）

- 容器 `max-w-[72rem]` 与网格对齐。
- 左/下：「← 返回工具库」链接（`--color-link`，`text-sm`）。
- 右/下：「同分类的其他工具」相关 1-2 卡片（用 EntryCard 渲染，排除当前 slug）。
- 相关工具由 lib 暴露 `getRelated(category, excludeSlug)`；不足时整区块 `hidden`。
- 上下分割线：`--color-border`。

### 档案页标题 + 计数（D-08）

- 分类档标题 Heading 24px display：`分类：{category}`；副标题 meta 13px 灰：`（{N} 个工具）`。
- 标签档标题：`标签：{tag}` + `（{N} 个工具）`。
- 标题上方面包屑。

### Empty state 组件

- 复用 Phase 1 Empty state 形态：居中、muted 底区块、`radius-xl`、`p-10`、装饰星/心 SVG（`aria-hidden`）+ heading + body + CTA。
- 三套文案见 Copywriting（主页 / 分类 / 标签各一套）。
- CTA 按钮样式同 Phase 1 次 CTA（`--color-surface` + `--color-border` 边框 + 文字 `--color-text`，hover primary）。

## Interaction & Motion（D-04 / TOOL-05）

| 交互 | 规格 |
|------|------|
| 卡片 hover 上浮 | `translateY(-2px)`、`duration-150`、`motion-safe:` guard；`prefers-reduced-motion: reduce` 下静止 |
| 整卡 href | 点击进入详情 `/tools/[slug]` |
| 外链按钮 href | `<a target="_blank" rel="noopener noreferrer">`，**不触发整卡跳转**（通过布局分离——整卡 `<a>` 不包裹按钮） |
| 外链事件 | 用原生 `<a>` 而非 JS；若用 `<button>` + click handler 则 `stopPropagation()` + `window.open(...,'_blank','noopener,noreferrer')`；**推荐纯 `<a>` 方案**（零 JS） |
| 分类 pill / tag chip | 整项可点，跳转静态档案页 |
| 返回 / 相关工具链接 | 整卡 `<a>` 跳转 |
| 装饰 | 漂浮星点 `deco-drift` keyframes（Phase 1 已定义），`prefers-reduced-motion` 禁用 |

- 本阶段**零 Preact / nanostores 岛屿**。所有交互由原生 `<a>` + CSS `:hover`/`:focus-visible` 处理。
- View Transitions 沿用 Phase 1 BaseLayout 配置即可；不新增 `astro:after-swap` 处理。

## Accessibility

| 项 | 契约 |
|----|------|
| 触控目标 | 整卡、外链按钮、分类 pill、面包屑链接 `min-h-11 min-w-11`（≥44px）；标签 chip ≥ 32px |
| 外链按钮 | `aria-label="{工具名} 外链"`，读屏明确指向外部；不使用通用「访问」 |
| 整卡可点 | 整卡为单一 `<a>`（`role` 默认 link），屏幕阅读器可聚焦并激活 |
| 焦点环 | `:focus-visible` 2px `--color-primary` + 2px offset，radius 跟随元素 |
| 装饰 | 空状态点缀、标签前 `#`、面包屑分隔符、计数括号均 `aria-hidden="true"` 或 `aria-hidden` |
| 分类 pill 激活态 | 当前分类页给 pill 加 `aria-current="page"` |
| 面包屑 | `<nav aria-label="路径">` + `<ol>`；当前项无链接 |
| 计数 meta | 计数文本对读屏有意义（不加 `aria-hidden`），「（N 个工具）」朗读自然 |
| 对比度 | Phase 01 已锁；暗/亮双主题正文 ≥ AA 4.5:1；`--color-link` 保持正文可用性 |
