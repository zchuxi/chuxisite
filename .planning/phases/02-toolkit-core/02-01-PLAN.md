# 02-01 · 数据层扩展（tags transform + lib 读取函数 + seed 同步）

## 目标

在 Phase 1 既有 `content.config.ts`（tools/anime 双层）与 `lib/tools.ts`（`getAllTools()` 收口）上做**最小扩展**，为后续卡片、详情、三档案页提供类型安全的 schema transform 标签归一化与可复用读取 API；同步更新既有 seed `raycast.mdx` 的 tags 为小写以与 transform 契约一致。

## 需求牵引

| Requirement | 本 plan 贡献 |
|---|---|
| TOOL-01 | `getAllTools()` 已是卡片网格主数据源（不变）；新增 `slugOf` 为卡片详情 `<a href>` 提供单一出口 |
| TOOL-03 | 新增 `getToolsByCategory(category)` — 分类档案页静态数据源 |
| TOOL-04 | schema `tags` transform 归一化（trim + lowercase + 去重）；新增 `getToolsByTag(tag)` 与 `getRelated(category, excludeSlug)` 支撑标签档案与详情页 related nav |

- `src/lib/anime.ts`（closest analog 风格参照）
</read_first>
<action>
在 `getAllTools()` 已有代码之后、文件末尾追加以下导出函数，**不重写 `getAllTools()`**：

1. `export function slugOf(title: string): string { return encodeURIComponent(title); }` — 单一 slug 派生出口，禁止在 components/pages 内联 `encodeURIComponent`。
2. `export async function getToolsByCategory(category: string)` — `(await getAllTools()).filter((t) => t.data.category === category)`。返回值沿用 zh title localeCompare 排序。
3. `export async function getToolsByTag(tag: string)` — 入参 `tag.toLowerCase()`（与 schema transform 的 lowercase 对齐），`(await getAllTools()).filter((t) => t.data.tags.includes(target))`。
4. `export async function getRelated(category: string, excludeSlug: string, n = 2)` — `(await getAllTools()).filter((t) => t.data.category === category && slugOf(t.data.title) !== excludeSlug).slice(0, n)`；返回值长度自然 ≤n，`-1` 取 1 时无整区由组件守卫。

**不导入 / 不引入 anime 集合**，lib 层改动仅位于 tools 文件。
</action>

- `npx astro check` 零 TS error（lib/tools.ts 类型通过）
</verify>
